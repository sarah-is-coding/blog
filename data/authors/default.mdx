Hi, I'm Sarah — a Software Engineer at [Drive Capital](https://drivecapital.com) based in Columbus, Ohio, where I work on Herbie, an internal platform that powers how startup investments are identified, evaluated, and managed across the firm.

I graduated from the University of Cincinnati with a Bachelor's in Computer Science in 2024. My path into software started with a robotics club in high school, where I spent a lot of time programming autonomous robots and realized that writing code to make things move and make decisions was exactly what I wanted to do.

## Experience

Before Drive Capital, I've worked across a pretty wide range of domains:

- **Booz Allen Hamilton** — Four co-op rotations working on signal processing algorithms deployed in airborne systems, using Python and Docker. Supported work requiring a U.S. DoD Secret clearance.
- **Asset Strategies Group** — Built retail planning and analytics software full stack: configurable analytics widgets, financial performance views, dependency-aware scheduling, and audit-tracked change history across UI, API, and database layers.
- **Nexgen Vending** — Freelance project engineering a Raspberry Pi kiosk app with Node.js for tracking arcade machine parts via QR code, with role-based access, automated low-stock alerts, and admin reporting.
- **Painless Analytics** — Co-op contributing to a web analytics startup, building RESTful routing and responsive interfaces with Laravel, PHP, and Bootstrap.

## Outside of Work

I did karate growing up, which taught me a lot about pushing through hard things. In high school I ran cross country and endurance races, and put a lot of hours into competitive robotics through VEX.

These days I'm still drawn to problems that require patience and iteration — whether that's debugging a gnarly race condition or figuring out why a migration broke production data.
